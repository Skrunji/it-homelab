# Day 5

Today I shifted away from databases and dashboards and focused on a core skill in IT automation, working directly with files using Python.

The goal was to simulate a real-world scenario: processing a CSV export of device inventory and turning it into something actionable.

I started by creating a small dataset representing device inventory, including asset tag, device type, assigned user, location, and status. From there, I built a Python script using pandas to read the CSV and load it into a DataFrame.

One of the first issues I ran into was a file path error. The script couldn’t find the CSV because it was running from the repo root while the file was in a subfolder. I fixed this by updating the file path in the script, which made it runnable from anywhere.

Once the data was loading correctly, I built out a series of reports.

First, I generated a summary of total devices by location using groupby and count. This mirrored SQL concepts I’ve already learned but applied them in pandas.

Next, I created a filtered report to identify devices needing attention which I defined as devices that are either unassigned or marked as needing repair. This required filtering the DataFrame using isin() and then grouping the results by location.

This step helped reinforce the importance of order of operations. Filtering must happen before grouping, otherwise the results don’t reflect the intended subset of data.

After that, I expanded the report to include actionable detail instead of just summary counts. I added a room column to the dataset and created a detailed output showing:

* Asset tag
* Assigned user
* Location
* Room

This made the report immediately usable in a real IT context instead of just knowing where problems exist, it shows exactly which devices need attention and where to find them.

Finally, I exported the filtered results to a new CSV file using method chaining:

`filtered[["asset_tag", "assigned_user", "room", "location"]].to_csv("csv-automation/devices_needing_attention.csv", index=False)`

This reinforced how pandas operations can be chained together to transform and output data efficiently.

By the end of the session, I had built a complete workflow:

* Read data from a CSV
* Process and summarize it
* Filter for meaningful conditions
* Output both summary and detailed reports
* Export actionable results for sharing

The biggest takeaway today was understanding the difference between summary data and actionable data. Counts are useful for awareness, but real value comes from knowing exactly what needs to be fixed and where.

This felt much closer to real IT work. Taking raw exports and turning them into something useful for decision-making and action.
