# Day 3

Today we're getting python installed in our environment and linking it to the sql server installed in the last session.

Here are the commands used in cmd:

```cmd
Winget search python. 
Winget install python.python.3.9.
exit
```
then in powershell:

```powershell
pip install mysql-connector-python
C:\Users\[username]\AppData\Local\Programs\Python\Python39\python.exe -m pip install --upgrade pip
```

Python wasn't working for me after multiple attempts trying to run commands in cmd, powershell and the built in terminal in VSCode. I tried adding the path manually through environment variables in windows and still was getting the error 'Python was not found; run without arguments to install from the Microsoft store, or disable this shortcut from Settings > Apps> Advanced app settings > App execution aliases."

So I ran Winget uninstall and then reinstalled python from the windows store.

when I attempted to run the python installer it said the legacy 'py' command was still installed. 

Python 3.9 was still installed despite winget saying it successfully uninstalled. 

so I manually uninstalled it through the GUI instead this time.

Got Python 3.14.3 installed and working this time.

but then get this error when I try to run pip install python-dotenv

```
Fatal error in launcher: Unable to create process using '"C:\Users\[USERNAME]\AppData\Local\Programs\Python\Python39\python.exe"  "C:\Users\[USERNAME]\AppData\Local\Programs\Python\Python39\Scripts\pip.exe" install python-dotenv': The system cannot find the file specified.
```

This was because pip still had a stale reference to the old Python 3.9 path that no longer exists. So to use the new python explicitly I instead ran this command:

```
python -m pip install python-dotenv
```
finally got dotenv installed which will allow me to keep certain variables in my repo private such as passwords and api keys.

when I installed dotenv a warning popped up in the terminal that says dotenv.exe is installed in 'C:\Users\[USERNAME]\AppData\Local\Python\pythoncore-3.14-64\Scripts' which is not on PATH Consider adding this directory to PATH.

So I went back to environment variables and cleaned up the old python 3.9 paths in there and added the path for dotenv.

I also updated the version of pip from 25.3 to 26.0.1 using this command:

```
python -m pip install --upgrade pip
```

After getting Python and the required libraries installed, I created a script called
`sakila-connect.py` in the Python folder of my repo. The script connects to the local
MySQL Sakila database using credentials stored in a `.env` file so that sensitive info
like passwords are never hardcoded in the repo.

Once connected, I ran a JOIN query to pull actor names and their film counts from the
database, the same query I had written in MySQL Workbench in the previous session. I
used the `pandas` library to load the results into a DataFrame which displays the data
in a clean, readable table format. Finally I exported the results to a CSV file using
`df.to_csv()`.

Key things learned today:
- How to use `.env` files and `python-dotenv` to manage credentials securely
- How `pandas` simplifies working with query results compared to raw cursor output
- How to export DataFrame results to CSV with a single line of code
